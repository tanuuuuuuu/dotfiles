# Task Digest スタイルガイド

`task-digest.sh` が Discord に投稿する内容のフォーマット規約。
**このガイドが Single Source of Truth**。スクリプト変更時は必ずこれを参照する。

## 設計原則

1. **スキャナビリティ優先**: 見て 3 秒で「何件・何が緊急か」が分かること
2. **最重要情報を先頭に**: 期日 > タイトル > 依頼者 > 要約
3. **視覚的階層**: 色 → 件数 → タイトル行 → 補足行 の順で目に入る
4. **コンパクト**: 1 issue = 最大 2 行 (タイトル行 + 補足行)
5. **色で緊急度を表現**: Discord embed の `color` フィールドを活用

## 全体構造（1 回のポストの中身）

Discord webhook の `embeds` 配列で複数 embed を 1 メッセージにまとめる。

```
[サマリー embed]   ← 全体集計とタイムスタンプ
[オーバーデュー]   ← 赤（あれば）
[期日近]          ← オレンジ（あれば）
[進行中]          ← 青（あれば）
[待ち]            ← 黄（あれば）
[その他]          ← グレー（あれば）
```

件数 0 のカテゴリの embed は出さない。

## サマリー embed

- `title`: `📋 タスク進捗 M/D (曜) HH:MM`（例: `📋 タスク進捗 5/5 (火) 13:00`）
- `color`: グレー（`#95A5A6` / 9807270）
- `description`: 各カテゴリの件数を 1 行ずつ。色付き丸絵文字で:
  ```
  🔴 オーバーデュー: 1件
  🟠 期日近 (3日以内): 0件
  🔵 進行中: 5件
  🟡 待ち / 停滞: 8件
  ⚪ その他: 0件
  ```
- `footer.text`: `合計 N 件`

件数 0 の行も載せる（全体の網羅性を示すため。本体 embed は省略しても、サマリーには表示）。

## カテゴリ embed

各カテゴリは 1 つの embed に集約。issue を `description` に詰める。

- `title`: カテゴリラベル + 件数（例: `🔴 オーバーデュー (1件)`）
- `color`: カテゴリ色（後述）
- `description`: issue リスト（1 issue 2 行、間に空行を 1 つ）

## Issue 1 件の書式

**2 行構成、間にインデントなし**:

```
**#1620 Yunon AdvCode 0件調査** · 5/8 (残3日)
　└ 松浦さん · W2 側で AdvCode マスタが未運用、原因切り分け中
```

- **1 行目**: `**#<番号> <タイトル>** · <期日表記>`
  - 番号は `<repo省略>#N` 形式（polaris→`p#1620`、workhub→`w#42`）
  - タイトルが長い場合は最大 60 文字、超過は `…` で打ち切り
  - リンクは `[**...**](URL)` 形式で全体をハイパーリンクにする
- **2 行目**: `　└ <依頼者> · <要約>`
  - 全角スペース + `└` で視覚的に下位情報を表現
  - 依頼者が body から取れなければ GitHub author（`@apokurinkansen` 等）
  - 要約は最大 80 文字、超過は `…`
  - 要約が空なら 2 行目自体省略可（依頼者だけ残す）
- 各 issue の間に空行 1 つ

## 期日表記ルール

| 状況 | 表記 |
|---|---|
| 期日超過 | `M/D (Nd経過)` 例: `5/2 (3d経過)` |
| 今日 | `M/D (今日)` |
| 1〜3日先 | `M/D (残Nd)` 例: `5/8 (残3d)` |
| 4日以上先 / 期日なし | 表示しない（記載しない） |

`Nd` の `d` は半角、数字も半角。年は省略（年跨ぎは無視 OK）。

## カテゴリ分類ルール

`task-digest.sh` の Python 部と整合させる:

| カテゴリ | 条件 | 色（HEX / DEC） |
|---|---|---|
| 🔴 オーバーデュー | 期日 < 今日 | `#E74C3C` / `15158332` |
| 🟠 期日近 | 今日 ≤ 期日 ≤ 今日+3日 | `#E67E22` / `15105570` |
| 🔵 進行中 | (期日なし or 4日以上先) かつ updatedAt ≤ 3日前 | `#3498DB` / `3447003` |
| 🟡 待ち / 停滞 | updatedAt ≥ 4日前 or labels に `blocked`/`waiting`/`待ち`/`on hold` | `#F1C40F` / `15844367` |
| ⚪ その他 | 上記いずれにも当てはまらない | `#95A5A6` / `9807270` |

判定は上から順、最初にマッチしたカテゴリを採用。

## ソート順

カテゴリ内で **createdAt 降順**（新しく作成された issue が上）。
理由: 直近で立てた issue ほど記憶に新しく扱いやすい。期日順だと古いタスクが上に来てかえって埋もれる。

## 文字数 / embed 数制限への対応

Discord の制限:
- 1 embed `description`: 4096 文字
- 1 message 全 embed 合計: 6000 文字
- 1 message: 最大 10 embeds

対応戦略:
1. 通常は **1 message に 6 embeds**（サマリー + 5 カテゴリ）でおさまる想定
2. カテゴリ embed が 4000 文字超えたら、その embed だけ複数 embed に分割（タイトルに `(続)` を付ける）
3. 全体が 6000 字超えたら **複数 message に分割**:
   - 1 通目: サマリー + オーバーデュー + 期日近
   - 2 通目: 進行中
   - 3 通目: 待ち + その他
4. 各 message 間で連続性が分かるよう、サマリー以外の embed の footer に `(N/M)` を付ける（複数通になった時のみ）

## 依頼者の取得ロジック

優先順位（最初にマッチしたものを採用）:

1. body 内の `## 報告元` `## 依頼元` `## 依頼者` `## 報告者` `## 相談者` セクションの 1 行目
2. body 内の `依頼:` `報告:` `相談:` `from:` の右側
3. GitHub の `author.login`（フォールバック）

抽出後の文字列は前後空白除去 + 連続空白を 1 つに圧縮 + 最大 40 文字。

## 要約の取得ロジック

優先順位:
1. body の `## 何をしたいか` セクションの先頭テキスト
2. body の `## 概要` セクションの先頭テキスト
3. body の `## 背景` セクションの先頭テキスト
4. body の先頭から（フォールバック）

抽出後の処理:
- コードブロック (` ``` ... ``` `) を除去
- 改行を半角スペースに変換
- 連続空白を 1 つに圧縮
- 最大 80 文字、超過は `…`

## やってはいけないこと

- ❌ Issue 1 件で 3 行以上使う
- ❌ 情報量ゼロの emoji（装飾だけの `✨` `🎉` 等）
- ❌ 同じ情報を別表現で 2 回（`期日: 5/8` と `あと 3 日` の併記）
- ❌ 各カテゴリに「合計」を書く（最後のサマリーだけ）
- ❌ `┌─ ─┐` 等の罫線（Discord で表示崩れる）
- ❌ Issue 行頭に `-` `*` `1.` 等を付ける（embed では空行区切りで十分）
- ❌ 太字を多用（タイトルだけに使う）

## やるべきこと

- ✅ カテゴリ embed のタイトルに件数を併記（`🔴 オーバーデュー (1件)`）
- ✅ 期日表記を統一形式 `M/D (...)` で揃える
- ✅ 件数 0 カテゴリは embed を省略（サマリーには載せる）
- ✅ ソート順を厳守（迷ったら期日昇順）
- ✅ 数字は半角
- ✅ 投稿失敗時は標準エラーに `[error]` 出力 + 終了コード非ゼロ
