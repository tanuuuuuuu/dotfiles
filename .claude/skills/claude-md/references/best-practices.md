# CLAUDE.md ベストプラクティス

## 構成の原則

### WHAT（何のプロジェクトか）

- 技術スタック（言語、フレームワーク、主要ライブラリ）
- プロジェクト構造（重要なディレクトリとその役割）
- 目的（1-2文で簡潔に）

### HOW（どう作業するか）

- ビルド/テストコマンド
- 開発時の注意点
- コード規約（linterで対応できないもののみ）

## アンチパターン

| 避けるべきこと | 理由 |
|---------------|------|
| コードスニペットの埋め込み | 古くなりやすい。ファイルパスで参照する |
| 詳細なセットアップ手順 | README.md に記載すべき |
| タスク固有の指示 | 毎回読み込まれるため無駄 |
| linterで対応可能な規約 | ツールに任せる |
| 過度な強調表現の乱用 | 本当に重要なものが埋もれる |

## チェックリスト

### 必須項目
- [ ] 技術スタックが明記されている
- [ ] 主要なディレクトリ構造がわかる
- [ ] ビルド/テスト方法が記載されている

### 品質項目
- [ ] 60行以下（理想）/ 300行以下（最大）
- [ ] コードスニペットを含んでいない
- [ ] README.md と重複していない
- [ ] 普遍的に適用される情報のみ

## 配置場所

| 場所 | 用途 |
|------|------|
| `プロジェクトルート/CLAUDE.md` | プロジェクト専用（git管理推奨） |
| `プロジェクトルート/CLAUDE.local.md` | 個人用（.gitignore） |
| `~/.claude/CLAUDE.md` | 全プロジェクト共通 |
