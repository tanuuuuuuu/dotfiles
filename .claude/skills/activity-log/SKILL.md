# Activity Log

Claude Code のセッション履歴をプロジェクト別に集約し、Markdown の箇条書きで出力するスキル。

## 使い方

ユーザーの自然言語の日付指定を解釈し、`scripts/activity_log.py` に渡す。

```bash
# 今日
python ~/.claude/skills/activity-log/scripts/activity_log.py

# 特定の日
python ~/.claude/skills/activity-log/scripts/activity_log.py 2026-03-11

# 期間指定
python ~/.claude/skills/activity-log/scripts/activity_log.py 2026-03-01 2026-03-05
```

## ワークフロー

1. ユーザーの発言から対象期間を特定する（「今日」「昨日」「今週」「先週」「3月」など）
2. 必要なら `date` コマンドで現在日時を取得し、正確な日付を算出する
3. `activity_log.py` を実行する
4. スクリプトの生データをもとに、下記の出力フォーマットに従って整形して表示する

## 出力フォーマット

スクリプトの出力はセッション単位の生データ（ユーザーとアシスタントの会話全文を含む）。
これを **トピック単位** に統合・要約してユーザーに見せる。
同じテーマのセッションは1つのトピックにまとめる（例: Ghostty 関連のセッションが3つあれば1トピックに統合）。

会話全文が含まれているので、何をしたか（概要）だけでなく、会話の中で得られた知見や結論（学び）も正確に抽出できる。

プロジェクトごとに以下の形式で出力する:

```
## <プロジェクト名>

[MM/DD HH:MM] トピック名
- 概要: 具体的な作業内容を1〜2文で
- 学び: 会話の中で得られた知見・結論・発見を1〜2文で

[MM/DD HH:MM] トピック名
- 概要: ...
- 学び: ...
```

最後に、全トピックの中から最も価値のある学びを1つ選んでピックアップする:

```
---

> **Today's Best Learning:** ここに一番の学びを書く
```

**Example:**

```
## dotfiles

[03/10 11:03] Ghostty 1.3.0 アップデート対応
- 概要: v1.2.3→1.3.0 で無効になった設定を調査し config を修正
- 学び: macos-non-native-fullscreen が廃止され window-decoration = false に移行

[03/10 11:43] Neovim 画像プレビュー
- 概要: image.nvim プラグインの導入を検討・設定
- 学び: Ghostty は Kitty graphics protocol に対応しており、追加設定なしで画像表示可能

[03/10 11:02] ESC キーで入力ソース切替
- 概要: ESC 押下時に自動で英数入力に切り替える設定を実装
- 学び: Karabiner-Elements の complex_modifications で IME 制御が可能。hidutil では不可

---

> **Today's Best Learning:** Ghostty は Kitty graphics protocol をネイティブサポートしており、Neovim の image.nvim と組み合わせるだけでターミナル内画像表示が実現できる
```

ルール:
- トピック名は短く（10〜20文字程度）、何をしたかが一目でわかるようにする
- 概要は会話内容をもとに、実際に行った作業と結果を具体的に書く
- 学びは会話の中の Claude の回答や結論から抽出する。単なる作業記録ではなく、知識として再利用できる情報を書く
- 学びが特にないセッション（単純な操作依頼など）は学びを省略してよい
- 同じテーマの複数セッションは1トピックにまとめる
- 「コミットして」「プッシュして」など単なる操作指示だけのセッションはトピックに含めない
- 「(セッション操作)」もスキップする
- 各プロジェクト内のトピックは時系列の古い順に並べる
- 複数セッションを1トピックにまとめた場合は、最初のセッションの時刻を使う
- **Today's Best Learning** は、今後の作業に活かせる汎用的な知見を優先して選ぶ

## 日付の解釈ガイド

| ユーザーの表現 | 引数 |
|---|---|
| 今日 / きょう | (引数なし) |
| 昨日 / きのう | 昨日の日付 |
| 今週 | 今週月曜 〜 今日 |
| 先週 | 先週月曜 〜 先週日曜 |
| 3/1〜3/5 | 2026-03-01 2026-03-05 |
| 3月11日 | 2026-03-11 |

日付が曖昧な場合は確認せずに最も自然な解釈を採用してよい。年は現在の年を使う。

## データソース

スクリプトは2段階でデータを取得する:

1. **sessions-index.json** (優先) — `summary` と `firstPrompt` フィールドを使用。インデックスが最新であれば正確
2. **JSONL ファイル** (フォールバック) — ファイル更新日時で絞り込み、中身のユーザーメッセージを抽出。インデックスが古い場合に使用

## 注意事項

- タイムゾーンは JST (Asia/Tokyo) 固定
- subagents のセッションは除外される
- セッション操作系のメッセージ (/clear 等) は「(セッション操作)」と表示される
