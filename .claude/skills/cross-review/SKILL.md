# Cross Review

PR を Codex と Claude の二段でレビューする。Codex の「新鮮な目」と Claude の「リポジトリ全体文脈の理解」を組み合わせ、ノイズを除いた信頼できる指摘リストを作る。

## いつ使うか

- 重要な PR で第三者視点が欲しいとき
- Codex 単独レビューだとハルシネや PR 意図の誤読があるため、Claude による検証を挟みたいとき

## ワークフロー

### 1. PR 特定

引数で PR 番号が指定されていればそれを使う。なければ `AskUserQuestion` で番号を確認。
リポジトリのパスもユーザーに確認（`~/repos/` 配下や `~/Desktop/`、`~/Documents/` などに散在することがある）。

### 2. PR 情報取得

```bash
cd <repo_path>
mkdir -p .claude/tmp
gh pr view <PR番号> --json number,title,author,body,baseRefName,headRefName,state,additions,deletions,changedFiles,url
gh pr diff <PR番号> > .claude/tmp/pr<番号>.diff
gh pr diff <PR番号> --name-only
```

一時ファイルは必ず `<repo>/.claude/tmp/` に保存する（グローバル方針）。`/tmp/` は使わない — リポジトリ外のため取り違えやレビュー対象の誤指定の原因になる。

PR 本文（`body`）から「PR の意図」「レビューしてほしいポイント」を抽出する。これが後の Codex プロンプトと Claude 検証の基準になる。

### 3. Codex で一次レビュー

`Agent` ツールで `codex:codex-rescue` サブエージェントを呼ぶ（スキル `codex:rescue` は使わない — スキルを経由するとセッションがハングする）。

プロンプトに含めるべき要素:
- PR 番号 / リポジトリ
- 差分ファイルのパス（`<repo>/.claude/tmp/pr<番号>.diff`）と変更ファイル一覧
- リポジトリのルートパス
- PR の意図（PR 本文から抽出）
- レビュー観点（PR 本文に「レビューポイント」がある場合はそのまま渡す）
- `~/.codex/AGENTS.md` の P0/P1/P2 ハイシグナル原則に従う指示
- 「ハルシネーション禁止。実差分とリポジトリの該当ファイルを読んでから指摘」の念押し

### 4. Claude による妥当性検証

Codex が返した各指摘について、以下を実コード・実スキーマで確認する:

- **事実誤認の有無**: 指摘の前提（「このカラムが存在しない」「この関数はこう動く」等）が実装と一致するか
- **PR 意図との整合**: PR 本文の方針と矛盾していないか。矛盾する場合、PR 意図が誤っているのか Codex が誤読しているのかを判定
- **指摘の射程**: Codex は差分しか見ていないため、リポジトリ全体の文脈で見ると過剰／不足な指摘になっていないか
- **優先度の妥当性**: P1 と分類されているがブロッカーレベル（P0）の場合、または逆の場合の補正

検証結果は以下の 3 分類:
- **採用**: 指摘内容そのまま反映
- **補正**: 指摘の方向性は正しいが内容を補正して反映（実装と齟齬がある部分を修正）
- **却下**: ハルシネ／誤読／既知問題／差分外の問題 など

### 5. 統合パンチリスト提示

ユーザーに以下のフォーマットで提示:

```
## クロスレビュー結果: PR #<番号>

### 採用（N 件）
- **[P?]** 指摘内容
  - Codex 指摘: <要約>
  - 検証: <実装と一致を確認した点>

### 補正（N 件）
- **[P?]** 指摘内容（補正版）
  - Codex 元指摘: <要約>
  - 補正理由: <実装と突き合わせた結果のズレ>
  - 採用案: <最終的な修正方針>

### 却下（N 件）
- Codex 指摘: <要約>
  - 却下理由: <ハルシネ / 誤読 / etc>

### 修正アクション
1. <ファイル>:<行> を <こう> 修正
2. ...
```

### 6. 修正適用

ユーザー承認後、提案されたアクションを順に実施。コミット・push まで進めるかは都度確認。

### 7. クリーンアップ

レビュー完了後、無条件で diff ファイルを削除する（累積防止）。

```bash
# macOS
trash .claude/tmp/pr<番号>.diff
# Linux/WSL
rm .claude/tmp/pr<番号>.diff
```

## 重要原則

- **Codex の指摘を鵜呑みにしない**: ハルシネ・差分外参照・PR 意図の誤読は一定の頻度で起きる。必ず実コードで検証
- **Claude も無謬ではない**: 自分の検証もまた誤る可能性がある。確信が持てない場合は「未確認」と明示してユーザーに判断を委ねる
- **PR 本文を一次情報として尊重**: PR 本人が「ここはこう判断した」と書いている部分は、Codex がそれと矛盾する指摘をしてきたら一旦 PR 意図を優先し、技術的問題があるなら別途明示する
- **ハイシグナル原則**: 確信がない指摘・スタイル提案・既存問題は出さない（`~/.codex/AGENTS.md` 準拠）

## 出力

最終アウトプットは「採用 / 補正 / 却下」の 3 分類されたパンチリストと、修正アクションリスト。
