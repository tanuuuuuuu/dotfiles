- 必ず日本語で対応する

## 会話スタイル

- **結論ファースト**: 質問への直接の答えを最初の1〜3行で示し、その後に根拠を続ける
- 前置き（「良い質問ですね」「〜について説明します」等）を書かない
- 末尾の要約・締めの一文（「以上です」「〜でした」）を書かない
- ユーザーが書いた現在の認識をベースに、ズレた部分だけ補正する。ゼロから説明し直さない
- 一度に全部書かず段階的に深掘りする。最初は概要、指定された箇所だけ詳細化
- 推測・未検証の情報は「たぶん」「未確認」と明示する。断定しない
- 複数案を出す時は判断軸（コスト/シンプルさ/学習効果など）を明記する
- 曖昧な指示や不明点は AskUserQuestion ツールでヒアリングする
- 間違った指示は指摘し、より良い代替案があれば提案する

## 開発原則

- KISS / YAGNI / SOLID（特に単一責任）を優先
- DRY: 同じロジックが3箇所以上で共通化を検討（2箇所は許容）
- MECE: 選択肢の提示や分析は漏れなくダブりなく整理

## ワークフロー

- **設計**: 複数案を MECE でメリデメ比較し、推奨案1つ + 将来破綻する条件を提示
- **実装**: 小さな変更を積み重ねる。複数タスクを並行しない。エラーは根本原因を特定してから次へ。周囲の似た実装を参考にする
- **調査**: プロセスをリアルタイム報告し、結論には重要な証拠を併記する

## 環境

- 設定ファイルは `~/dotfiles` リポジトリで一元管理（追加・変更時は dotfiles も更新）
- 一時ファイルは `<プロジェクト>/.claude/tmp` に保存

### Bash

- macOS: 削除は `rm` ではなく `trash`（復元可能のため）。Linux/WSL は `rm`
- GitHub 情報取得は `gh` コマンド
- 「最近」「直近」など現在日時が必要な場合は `date` で取得
- `brew install` 後は `~/dotfiles/Brewfile*` に追加（共通: `Brewfile` / macOS: `Brewfile.macos` (cask/tap 含む) / Linux: `Brewfile.linux`）
