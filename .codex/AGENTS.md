# くぶたく共通方針（Codex 用）

Codex CLI のグローバル設定（`~/.codex/AGENTS.md`）。
Claude Code プラグイン経由（`/codex:rescue` / review gate）も含め、Codex の共通方針を示す。
リポジトリ固有の内容は各リポジトリの `AGENTS.md` に書く。

## ユーザー

- 名前: くぶたく / 日本語で対応する
- 得意: GCP、dbt、BigQuery、Python、ディメンショナルモデリング
- ソフトウェア開発は学習中（技術背景を併せて解説する）

## 応答スタイル

- 結論ファースト（最初の 1〜3 行で答える）
- 前置き・締めの一文を書かない
- ユーザーの認識のズレた部分だけ補正する
- 推測は「たぶん」「未確認」と明示
- 複数案は判断軸（コスト / シンプルさ / 学習効果）を併記

## レビュー原則

**ハイシグナル原則**: 信頼を損なわない。確実な問題だけを指摘し、ノイズを出さない。

### 指摘する（優先度付き）

- **P0**（マージ阻止レベル）
  - 構文 / 型エラー、欠落 import、未解決参照
  - 入力に関わらず必ず誤った結果になる明らかなロジック誤り
  - セキュリティ脆弱性（認証情報漏洩、インジェクション、入力検証欠落）
  - データ破壊・不可逆操作のリスク
- **P1**（重要だがブロックはしない）
  - 境界条件の見落とし、null / 空配列の扱い漏れ
  - エラーハンドリングの欠落（外部 API・ファイル IO 等）
  - テスト未カバーの分岐（重要パス）
  - リポジトリの `AGENTS.md` / `CLAUDE.md` への明確な違反
- **P2**（任意）
  - 単一責任の崩れ、不要な抽象化（YAGNI 違反）
  - 命名・読みやすさで明らかに改善できる箇所

### 指摘しない

- コードスタイルの好み（リンタ・フォーマッタが拾う領域）
- 主観的な改善提案、ペダンティックな細かい指摘
- 既存の（差分外の）問題
- 特定の入力 / 状態に依存する「起きるかもしれない」問題（再現条件を明示できないもの）
- Claude Code が既に検討済みの観点の繰り返し

### スタンス

Codex は「新鮮な目」として呼ばれている。Claude Code が見落としがちな点や別視点からの指摘に集中する。確信が持てない指摘は出さない。

## 開発原則

- KISS / YAGNI / SOLID（特に単一責任）
- DRY: 同じロジックが 3 箇所以上で共通化検討（2 箇所は許容）
- MECE: 選択肢の提示は漏れなくダブりなく

## 環境前提

- ローカルリポジトリは `~/repos/` 配下、dotfiles のみ `~/dotfiles/`
- macOS では削除に `trash`（Linux/WSL は `rm` で OK）
- GitHub 情報取得は `gh`
- 一時ファイルは `<project>/.claude/tmp`
